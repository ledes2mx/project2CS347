How I met requirements!

I met the requrements set out by the assignment by...

  • Not having a to-do list as my project.
  • Using MySQL as my database, so that I didn't have to use MongoDB or NoSQL.
  • Using an express based service to interact with the database.
  • Having the endpoints appropriately named.
  • Having the service send and recieve complex data as JSON.
  • Supporting the full CRUD operation spectrum.
  • Having the database only be accessable in the droplet, and blocking all ports except the ones needed.
  • Running Nginx to allow encripted access to the web service.
  • Supporting cross-origin resource sharing using middleware and an options wildcard endpoint.
  • Running the web service with pm2 so it stays running.
  • Having the service stored on Github and is shared.
  • Having the database's creation and schema on the repository.
  • Commiting and pushing steadily over the process.
  • Having my service respond to https://project2.miguelcodessometimes.me:8443/report.html

These are the parameters for each thing

For the GET requests, you can either..

  • request https://project2.miguelcodessometimes.me:8443/log
  • request https://project2.miguelcodessometimes.me:8443/log/:day/:month/:year

Day is a number, month is a string, year is a number.

For posting, you must have the day(a number), the month(a string), a year( a number), a temperatrue(as temp(a number)), and the weather(any text).

For a patch request, you must do https://project2.miguelcodessometimes.me:8443/log/:id where :id is a number that is shown in the get request. You must have all the parameters above in the post to see the change.

For the delete request, all you have to have to delete something is the id. So you would use https://project2.miguelcodessometimes.me:8443/log/:id to delete an entry.
